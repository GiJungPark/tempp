# 홈택스 문서 인덱스

이 폴더는 홈택스 연동 작업을 다시 이어받는 사람이 필요한 문서를 빠르게 찾도록 구성했다.

## Obsidian Vault 기준

옵시디언에서는 아래 폴더를 vault root로 열면 `docs/` 구현 문서와 `source/` 원문 자료가 함께 연결된다.

```text
/Users/nox/Documents/홈택스
```

## 문서 성격

| 폴더 | 성격 | 언제 읽나 |
| --- | --- | --- |
| `source` | 기획서, 전산매체 원문, 샘플 파일 | 원문 기준 확인이 필요할 때 |
| `원문자료` | `source` 파일로 이동하기 위한 지도 | 원문 위치를 찾을 때 |
| `기획` | 기획서 정책을 구현 관점으로 요약 | 입력 모델/계산/신고 정책을 볼 때 |
| `스크래핑` | 홈택스 로그인, 세션, 내부 API, 파일 업로드/검증/제출 흐름 | 홈택스 API 호출 로직을 만들 때 |
| `전자자료` | 홈택스에 올릴 고정폭 전자파일 생성 규격 | 원천세/간이지급명세서 파일 generator를 만들 때 |

## 주의

- `전자자료/전자파일 생성 규격 전체.md`는 레거시 링크 호환용 목차다.
- 실제 구현 문서는 분리된 업무별 문서를 기준으로 읽는다.
- `source/`는 원문 보관소이고, `docs/`는 구현자가 바로 읽는 요약/설계 문서다.
- 간이지급명세서 API의 일부 코드값은 아직 HAR로 확정이 필요하다.
