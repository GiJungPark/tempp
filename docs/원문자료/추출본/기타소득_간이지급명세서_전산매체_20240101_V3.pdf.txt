잔신매체 작성방법
가. 전산매체 파일 구성
○ 전산매체로 제출하는 간이지급명세서( 거주자의 기타소득) 제출 파일은 A,B,C 3개의 레코드로
구성됩니다.
○ A, B, C의 각 레코드 길이는 170 바이트로 모두 동일한 길이로 구성되어야 합니다.
레코드 유형 내 용 길이 비 고
A 레코드 제출자 레코드 170 바이트
B 레코드 지급자 별 집계 레코드 170 바이트
C 레코드 소득자 레코드 170 바이트
○ 전산매체의 파일명은 소득종류(파일명구분자)와 제출자의 사업자등록번호를 연결하여 사용합니다.
- 간이지급명세서(거주자의 기타소득) : SE
※ 예 시 : 사 업 자 등 록 번 호 가 ‘123-45-67890’ 인 간 이 지 급 명 세 서 ( 거 주 자 의 기 타 소 득 ) 의
파일명은 ‘ SE1234567.890 ’ 임
나. 레코드 설명
○ A 레코드(제출자 레코드)
- 간이지급명세서(거주자의 기타소득) 자료 제출자에 대한 정보를 수록한 레코드로서
- 자료 제출자의 사업자등록번호, 제출하는 간이지급명세서의 지급자 수 등을 수록하며,
- 제출파일 1개당 반드시 1개의 레코드를 수록해야 합니다.
○ B 레코드(지급자 별 집계 레코드)
- 지급자에 대한 정보 등을 수록한 레코드로서
- 지급자의 사업자등록번호 등과 제출하는 자료의 건수, 금액 등의 집계를 수록하며,
- 제 출 파 일 에 수 록 된 지 급 자 별 로 1 개 의 레 코 드 를 수 록 해 야 합 니 다 . 즉 , A 레 코 드 에 수 록 된
지급자 수 만큼 B 레코드를 수록해야 합니다.
○ C 레코드(소득자 레코드)
- 소득을 지급받은 자(거주자의 기타소득자)에 관한 레코드로서
- 기타소득을지급받은자의 인적사항, 소득구분,지급액,소득세,지방소득세등을수록하며,
- 지급자별로 귀속연도 별-귀속월 별-소득자 별-소득구분 별 수 만큼 C 레코드를 수록해야 합니다.
＊ 소득자에게 귀속연도, 귀속월, 소득구분이 다른 기타소득을 각각 지급한 경우 각각 C레코드를 수록하여야 하며, 이 경우 소득자 인원수 산정 시에는
각각을 별도의 소득자로 간주 합니다
(예시) 홍길동에게 강연료 등(76)과 자문료 등(79)에 대한 기타소득을 지급한 경우 각각의 소득구분 별로 C 레코드에 2건 생성하며, 소득자 인원수도 2명으로 계산
