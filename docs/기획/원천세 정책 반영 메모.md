## 확정 정책

| 영역 | 정책 |
| --- | --- |
| 대상 | 사업소득 또는 기타소득을 지급하는 개인사업자 |
| 지원 신고 | 정기신고 `01`, 기한후신고 `03` |
| 미지원 신고 | 수정신고 `02`, 원천세 반기 신고 |
| 지급월 | 화면 상단 공통 선택, 기본값은 진입월의 전월 |
| 귀속월 | 행별 선택, 지급월보다 늦을 수 없음 |
| 제출월 | 원천징수이행상황신고서의 제출연월은 지급월 + 1 |
| 지급월 상한 | 1~11월 귀속은 같은 해 12월 지급까지, 12월 귀속은 다음해 2월 말일 지급까지 |
| 소득자 전화번호 | 원천세/명세서 제출에 불필요하므로 수집하지 않음 |
| 사업장 정보 | 홈택스 사업장 정보 조회 결과를 사용 |
| 외국인 | 거주자만 지원, 내외국인구분에 내국인 `1`/외국인 `9` 반영 |
| 위택스 | 홈택스 원천세 신고 후 지방소득세 신고 완료 여부를 별도로 안내/확인 |

## 입력 모델

신고 세션 단위:

```text
사업장
지급월
신고 기준일
사업소득 행 N개
기타소득 행 N개
```

소득자 행 공통:

```text
소득자 이름
외국인 여부
식별번호: 내국인 주민등록번호 / 외국인 외국인등록번호 또는 여권번호
귀속연월
금액 입력 기준: 세전 / 세후
입력 금액
```

사업소득 행:

```text
업종코드
```

기타소득 행:

```text
소득구분코드: 76 또는 79
```

## 신고 모드 판정

정기신고 마감일 `D`:

```text
D = 지급월의 다음달 10일
단, 토요일/일요일/공휴일/근로자의 날이면 다음 영업일로 순연
```

판정:

| 조건 | 신고구분상세코드 | 가산세 |
| --- | --- | --- |
| 오늘 <= 해당 지급월의 D | `01` 정기 | 없음 |
| 오늘 > 해당 지급월의 D | `03` 기한후 | 납부지연 가산세 |

주의:

- 과거 누락분은 지급월별 D를 기준으로 기한후 여부를 판정한다.
- source 기획서 ①은 “정기 기간에도 과거분 기한후 가능”이라고 적고, source 기획서 ④는 “정기 기간에는 기한후를 받지 않음”이라고 적어 충돌한다. 구현 전 [[docs/기획/원천세 정책 반영 메모#확인 필요|확인 필요]]를 봐야 한다.

## 사업소득 계산

세전 입력:

```text
원천세 = floor(세전금액 * 0.03)
지방소득세 = floor(원천세 * 0.1)
세후금액 = 세전금액 - 원천세 - 지방소득세
```

세후 입력:

```text
원천세 = floor(세후금액 * 0.03 / 0.967)
지방소득세 = floor(원천세 * 0.1)
세전금액 = 세후금액 + 원천세 + 지방소득세
```

소액부징수:

```text
소득자별 원천세가 1,000원 미만이면 원천세 0원
```

## 기타소득 계산

서비스 지원 코드:

| 코드 | 내용 | 필요경비 |
| --- | --- | --- |
| `76` | 인적용역 중 그외, 강연료 등 | 60% |
| `79` | 인적용역 중 자문료 | 60% |

세전 입력:

```text
필요경비 = 세전금액 * 필요경비율
소득금액 = 세전금액 - 필요경비
원천세 = floor(소득금액 * 0.2)
지방소득세 = floor(원천세 * 0.1)
세후금액 = 세전금액 - 원천세 - 지방소득세
```

세후 입력, 필요경비 60%:

```text
원천세 = floor(세후금액 * 0.08 / 0.912)
지방소득세 = floor(원천세 * 0.1)
소득금액 = 세후금액 + 원천세 + 지방소득세
필요경비 = 소득금액 * 0.6 / 0.4
세전금액 = 소득금액 + 필요경비
```

과세최저한:

```text
소득금액 <= 50,000원이면 원천세/지방소득세 0원
```

판정 순서:

```text
1. 기타소득 과세최저한
2. 소액부징수
3. 일반 원천징수
```

## 전자파일 생성 반영

원천세:

- `21` Header의 귀속연월은 행의 귀속연월 또는 신고 단위 귀속연월 정책에 맞춘다.
- `21` Header의 지급연월은 신고 세션 지급월이다.
- `21` Header의 제출연월은 지급월 + 1이다.
- `23` 레코드는 소득유형별 집계로 생성한다.
- 사업소득은 `A25`, 가감계 `A30`을 우선 생성한다.
- 기타소득은 `A42`, 가감계 `A40`을 우선 생성한다.
- 전체 합계가 필요하면 `A99`를 생성한다.

간이지급명세서:

- 사업소득과 기타소득은 파일을 분리한다.
- 사업소득 파일명은 `SF{사업자등록번호 앞 7자리}.{사업자등록번호 뒤 3자리}`다.
- 기타소득 파일명은 `SE{사업자등록번호 앞 7자리}.{사업자등록번호 뒤 3자리}`다.
- A/B/C 레코드 집계 무결성을 자체 검증한다.

예:

```text
123-45-67890 -> SF1234567.890
123-45-67890 -> SE1234567.890
```

## 확인 필요

| 항목 | 충돌/미확정 내용 | 현재 권장 |
| --- | --- | --- |
| 과거분 기한후 입력 | 기획서 ①은 정기 기간에도 과거 지급월 기한후 허용, ④는 정기 기간에는 기한후를 받지 않음 | 지급월별 D 기준으로 판정하는 ①을 우선하고 ④ 문구 수정 요청 |
| 외국인 여권번호 | ①은 국적코드 저장하지 않음, ④는 여권번호+국적코드라고 표현 | 전산매체 원문 필드 기준으로 국적코드 필요 여부 재확인 |
| 세후 역산 보정 | 공식은 있으나 절사 후 1원 차이 보정 정책 없음 | 계산 결과를 표시하고 세전금액을 최종 기준으로 저장할지 결정 필요 |
| 기한후 가산세 배분 | 납부지연 가산세 공식은 있으나 전산매체 `23` 레코드 배분 방식 미정 | 소득코드별 세액 비율 배분 또는 합계 코드 반영 방식 확정 필요 |
| 원천세 귀속연월 | 원천세 Header는 단일 귀속연월 필드인데 기획은 행별 귀속월을 허용 | 한 신고 파일에 여러 귀속월 행을 섞을 수 있는지 홈택스 검증 필요 |
