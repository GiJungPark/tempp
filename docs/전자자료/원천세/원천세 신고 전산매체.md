# 원천세 신고 전산매체

## 목적

원천징수이행상황신고서 전자파일 generator 구현에 필요한 레코드 구조를 정리한다.

공통 padding/인코딩 규칙은 [[docs/전자자료/공통 전자파일 생성 규칙]]을 먼저 따른다.

## 원문과 서비스 입력 매핑

원천세 전자파일 생성의 기준은 `source/전산매체`에 보관한 국세청 전산매체 제출요령 원문이다.

- [[source/전산매체/전산매체/원천세 신고 전산매체 2faae3b52583802b8edec7c6969799e9|source 원천세 신고 전산매체 정리본]]
- [[docs/원문자료/추출본/원천세이행상황신고서_전산매체제출요령_20260102.doc.txt|원천세 DOC 추출본]]
- [[docs/원문자료/추출본/원천세이행상황신고서_전산매체제출요령_20260102.hwp.txt|원천세 HWP 추출본]]

사용자가 정리한 표는 기획/서비스 로직 관점의 매핑 자료다.

```text
원문 제출요령: 필드 길이, 레코드 구조, 코드표, 파일명, 작성 유의사항의 기준
서비스 표: 우리 서비스 입력값을 어느 필드에 넣을지 결정하는 매핑 기준
```

따라서 구현 시 원문 제출요령을 우선하고, 서비스 표의 `입력!!!` 값은 default/mapping 규칙으로 사용한다.

## 파일명 규칙

원문 제출요령의 “파일명을 부여하는 방법” 기준:

```text
일자 + 원천징수이행상황신고서 서식코드(C103900).신고구분상세코드
```

예:

```text
20210207C103900.01
```

현재 샘플:

```text
20260213C103900.01
```

생성식:

```ts
const fileName = `${dateYmd}C103900.${reportDetailCode}`;
```

여기서 `reportDetailCode`는 Header 6번 필드인 신고구분상세코드와 동일하게 맞춘다.

## 파일 구조

원문 제출요령에는 원천징수이행상황신고서 본문뿐 아니라 부표, 환급신청서, 승계명세서, 기납부세액명세서, 전월미환급세액 조정명세서까지 포함되어 있다.

전체 레코드 카탈로그:

| 번호 | 서식코드 | 서식명 | 자료구분 | 길이 | 비고 |
| --- | --- | --- | --- | --- | --- |
| 1 | `C103900` | 원천징수이행상황신고서 Header | `21` | 400 | 기본 |
| 2 | `C103900` | 원천징수이행상황신고서 환급세액 조정 | `22` | 200 | 기본 |
| 3 | `C103900` | 원천징수이행상황신고서 원천징수 명세 및 납부세액 | `23` | 150 | 소득코드별 반복 |
| 4 | `C103901` | 원천징수이행상황신고서 부표 | `24` | 150 | 필요 시 |
| 5 | `C103902` | 원천징수세액환급신청서 부표 | `27` | 215 | 환급신청 시 |
| 6 | `C103905` | 합병 및 사업자단위과세 전환 등에 따른 차월이월환급세액 승계명세서 | `26` | 100 | 승계명세 시 |
| 7 | `C103903` | 기납부세액명세서 원천징수신고납부현황 | `27` | 130 | 기납부세액명세서 |
| 8 | `C103903` | 기납부세액명세서 지급명세서 기납부세액 현황 | `28` | 200 | 기납부세액명세서 |
| 9 | `C103903` | 기납부세액명세서 기납부세액 차이조정현황 | `29` | 400 | 기납부세액 차이 시 |
| 10 | `C103904` | 전월미환급세액 조정명세서 전월미환급세액내역 | `27` | 100 | 환급신청 + 전월미환급 |
| 11 | `C103904` | 전월미환급세액 조정명세서 환급세액 조정현황 | `28` | 150 | 환급신청 + 전월미환급 |

현재 서비스의 최소 신고 케이스는 아래 3종 레코드만 우선 생성한다.

```text
21 Header 레코드                  400 byte
22 환급세액 조정 레코드           200 byte
23 원천징수 명세 및 납부세액      150 byte, 소득코드별 반복
```

레코드 예:

```text
21C103900...
22C103900...
23C103900A25...
23C103900A30...
```

단, 환급신청, 부표 대상 소득, 차월이월환급세액 승계, 기납부세액명세서 제출 대상이면 추가 레코드가 필요하다.

## 생성 판단 요약

### 항상 생성

```text
21 Header
22 환급세액 조정
23 원천징수 명세 및 납부세액: 실제 금액이 있는 소득코드만
```

### 조건부 생성

| 레코드 | 생성 조건 |
| --- | --- |
| `24 C103901` 원천징수이행상황신고서 부표 | 부표 작성 대상 소득이 있는 경우 |
| `27 C103902` 원천징수세액환급신청서 부표 | 정기신고에서 환급신청서를 첨부하는 경우 |
| `26 C103905` 차월이월환급세액 승계명세서 | 합병/사업자단위과세 전환 등 승계명세가 있는 경우 |
| `27/28/29 C103903` 기납부세액명세서 | 기납부세액명세서를 제출하는 경우 |
| `27/28 C103904` 전월미환급세액 조정명세서 | 환급신청 시 전월미환급세액이 있는 경우 |

### 생성하면 안 되는 경우

- 금액이 없는 소득코드 레코드를 `0`으로 채워 생성하면 안 된다.
- 원천징수이행상황신고서 부표도 실제 금액이 입력된 소득에 대해서만 생성한다.
- 원천징수세액환급신청서 부표도 환급이 발생한 소득에 대해서만 생성한다.
- 수정신고/기한후 신고에는 환급신청서를 첨부할 수 없다.
- 수정신고/기한후 신고에는 기납부세액명세서를 첨부할 수 없다.

## Header 주요 작성 규칙

### 신고구분

```text
신고구분코드: 01 정기
신고구분상세코드: 01 정기, 02 수정, 03 기한후
신고서종류코드: F01
세목코드: 14 원천신고
```

민원종류코드:

```text
FF001 = 원천징수이행상황신고서 / 원천징수세액환급신청서 / 기한후 신고서
FF101 = 원천징수이행상황 수정신고서
```

### 귀속연월 / 지급연월 / 제출연월

정기신고서:

```text
제출연월 = 지급연월 + 1
매월신고: 지급연월 = 제출연월 - 1
지급연월 >= 귀속연월
소득처분=N이면 지급연월의 연도 = 귀속연월의 연도
```

source 기획서 기준으로 “제출연월”은 실제 업로드 날짜의 연월이 아니라 해당 신고서의 귀속 제출월이다. 과거 지급월을 기한후로 늦게 올리더라도 제출연월은 지급월 + 1로 만든다.

반기신고 계속사업자:

```text
상반기: 귀속연월 1월, 지급연월 6월, 제출연월 7월
하반기: 귀속연월 7월, 지급연월 12월, 제출연월 다음연도 1월
```

반기포기/폐업사업자:

```text
귀속연월 = 반기 시작월
지급연월 = 반기포기 신청일 또는 폐업일이 속하는 월
제출연월 = 상반기 7월, 하반기 다음연도 1월
```

수정/기한후 신고서:

```text
source 기획서 기준: 제출연월 = 지급연월 + 1
기한후신고 귀속연도는 현재연도 5년 전까지 가능
수정신고 귀속연도는 5년 전까지 가능
소득처분=N이면 지급연월의 연도 = 귀속연월의 연도
```

### Header Y/N 플래그

| 필드 | 기본 | Y 조건 |
| --- | --- | --- |
| 원천신고구분 | `01` | 매월 `01`, 반기 `02` |
| 연말정산여부 | `N` | 세부 원천징수소득코드에 `A04`, `A05`, `A06`, `A26`, `A46` 중 하나라도 포함 |
| 소득처분여부 | `N` | 법인의 소득처분 내역을 신고 |
| 환급신청여부 | `N` | 정기신고에서 환급신청서를 첨부 |
| 일괄납부여부 | `N` | 본점일괄 납부승인 사업자 |
| 사업자단위과세여부 | `N` | 사업자단위과세 대상자 |
| 신고서부표여부 | `N` | 부표 작성 대상 소득 존재 |
| 차월이월환급세액 승계명세여부 | `N` | 승계명세서 첨부 |
| 기납부세액명세서 제출여부 | `N` | 기납부세액명세서 첨부 |

주의:

- 수정/기한후 신고 시 환급신청여부를 `Y`로 할 수 없다.
- 수정/기한후 신고 시 기납부세액명세서를 첨부할 수 없다.
- 환급신청액이 5,000만원 미만인 경우에만 은행코드/계좌번호를 기재한다.
- 은행코드와 계좌번호는 둘 다 있거나 둘 다 없어야 한다.

## 환급세액 조정 작성 규칙

- `당월조정환급세액계`는 각 소득의 가감계 항목에 작성된 당월조정환급세액 합계를 기재한다.
- 정기/기한후 신고에서는 당월조정환급세액계에 양수만 허용된다.
- 수정신고에서 추가납부금액 조정을 위해 A99 당월조정환급세액이 음수인 경우, 환급세액 조정 레코드의 당월조정환급세액계는 `0`으로 기재한다.
- 환급신청액은 환급신청서를 첨부한 경우 환급신청 총금액을 기재한다.
- 수정/기한후 신고 시 환급신청은 불가하므로 환급신청액은 `0`으로 기재한다.
- 승계대상합계 차월이월환급세액은 `C103905` 승계명세서를 첨부하는 경우 해당 서식의 차월이월환급세액 합계를 기재한다.

## 원천징수 명세 작성 규칙

- 원천징수소득코드는 지급연월 기준으로 코드표의 적용시작일자/적용종료일자를 검증한다.
- 인원과 총지급액은 양수 검증 대상이다.
- 인원이 0인데 총지급액이 있으면 오류.
- 인원이 1명 이상인데 총지급액이 0 이하이면 오류.
- 인원이 총지급액보다 크거나 같으면 오류.
- 1인당 총지급액이 1,000억원 이상이면 오류.
- 인원이 50만명 이상이면 오류.
- 총지급액이 10조 이상이면 오류.
- 실제 금액이 입력된 소득에 대해서만 레코드를 생성한다.
- 금액이 없는 소득코드를 `0`으로 채운 레코드로 생성하면 안 된다.

음수 허용:

```text
징수세액(소득세 등): A02, A04, A06, A20, A26, A28, A46, A90은 음수 가능
징수세액(농특세): A02, A04, A06, A26, A28, A90은 음수 가능
당월조정환급세액: 수정신고서는 양수/음수 가능
```

가감계:

```text
A99는 양수금액의 계 또는 수정신고의 양수/음수 가감계를 기재한다.
A04/A05/A06은 A10 근로소득 가감계에 반영한다.
A26/A27/A28은 A30 사업소득 가감계에 반영한다.
반기별 신고 또는 기한후 신고 시 A05, A27 분납금액은 입력할 수 없다.
```

## 추가 레코드 구현 메모

### `24 C103901` 원천징수이행상황신고서 부표

- 자료구분 `24`, 서식코드 `C103901`, 길이 150.
- 부표 원천징수소득코드 `C01`, `C02` 등은 별도 코드표를 따른다.
- 지급연월 기준으로 코드 적용시작/종료일자를 검증한다.
- 실제 금액이 입력된 소득에 대해서만 생성한다.

### `27 C103902` 원천징수세액환급신청서 부표

- 정기신고서에서만 첨부 가능하다.
- 수정/기한후 신고 시 첨부할 수 없다.
- 환급이 발생한 소득에 대해서만 작성한다.
- 합계 항목은 작성하지 않는다.
- 환급신청액은 환급세액 조정 레코드의 환급신청액과 일치해야 한다.
- 전월미환급세액이 1 이상이면 최초 발생시점 내역을 전월미환급세액 조정명세서에 작성해야 한다.

### `26 C103905` 차월이월환급세액 승계명세서

- 합병, 사업자단위과세 전환 등으로 차월이월환급세액을 승계할 때 작성한다.
- 차월이월환급세액 승계근거는 `01`, `02`, `03` 코드로 작성한다.
- 수정/기한후 신고 시에도 첨부 가능하다.
- 변동사항이 없는 자료도 수정분으로 작성한다.

### `27/28/29 C103903` 기납부세액명세서

- 수정/기한후 신고서에는 첨부할 수 없다.
- 원천징수신고납부현황의 합계 항목은 작성하지 않는다.
- 지급명세서 기납부세액 현황의 합계 항목은 작성하지 않는다.
- 원천징수신고납부현황과 지급명세서 기납부세액 현황의 소득종류가 일치해야 한다.
- 차이금액이 0이 아니면 차이조정현황의 사유코드 중 하나 이상이 `Y`여야 한다.
- 사유코드가 기타인 경우 사유내용은 필수다.

### `27/28 C103904` 전월미환급세액 조정명세서

- 환급신청 시 전월 미환급세액이 있는 경우 작성한다.
- 수정/기한후 신고서에는 첨부할 수 없다.
- 최초 발생시점은 귀속년월, 지급년월, 신고구분 기준이다.
- 같은 최초 발생시점에 여러 소득에서 환급이 발생하면 환급 발생 소득 종류만큼 내역을 작성한다.
- 전월미환급세액내역의 합계 항목은 작성하지 않는다.
- 환급세액 조정현황 마지막 줄의 차월이월환급세액은 원천징수이행상황신고서 환급세액조정의 전월미환급세액과 일치해야 한다.
