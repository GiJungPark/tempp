# 사업소득 간이지급명세서 전산매체

관련 문서: [[docs/전자자료/간이지급명세서/index|간이지급명세서 전자자료 인덱스]], [[docs/전자자료/공통 전자파일 생성 규칙]], [[docs/기획/원천세 정책 반영 메모]], [[docs/전자자료/간이지급명세서/기타소득 간이지급명세서 전산매체]], [[docs/스크래핑/간이지급명세서 API 분석]], [[source/전산매체/전산매체/사업소득 간이지급명세서 전산매채 2faae3b52583809f810ef113f5b80fa5|source 사업소득 간이지급명세서 정리본]]

## 목적

간이지급명세서(거주자의 사업소득) 전자파일 generator 구현에 필요한 A/B/C 레코드 구조를 정리한다.

공통 padding/인코딩 규칙은 [[docs/전자자료/공통 전자파일 생성 규칙]]을 먼저 따른다.

## 파일 구조

모든 레코드 길이: 170 byte

```text
A 레코드: 자료제출자 1건
B 레코드: 지급자별 집계, 지급자 수만큼 반복
C 레코드: 소득자별 상세, 지급자별 소득자 수만큼 반복
```

현재 테스트 기준은 제출자 = 지급자 1명이다.

```text
A 1건
B 1건
C N건
```

자료구분:

```text
50
```

파일명:

```text
SF{사업자등록번호 앞 7자리}.{사업자등록번호 뒤 3자리}
예: 123-45-67890 -> SF1234567.890
```

source 기획서 기준 소득자 전화번호는 전자파일 생성에 사용하지 않는다. A13 담당자 전화번호는 자료제출 담당자/고객 전화번호 필드다.

## A 레코드: 자료제출자

레코드 길이: 170 byte

| 순번  | 필드         | 타입  | 길이  | 누적  | 값/생성 규칙                   |
| --- | ---------- | --- | --- | --- | ------------------------- |
| A1  | 레코드 구분     | X   | 1   | 1   | `A`                       |
| A2  | 자료구분       | 9   | 2   | 3   | `50`                      |
| A3  | 세무서코드      | X   | 3   | 6   | 제출자 소재지 관할 세무서 코드         |
| A4  | 제출연월일      | 9   | 8   | 14  | 오늘 `YYYYMMDD`             |
| A5  | 제출자 구분     | 9   | 1   | 15  | 개인 `3`, 법인 `2`, 세무대리인 `1` |
| A6  | 세무대리인 관리번호 | X   | 6   | 21  | 세무대리인이 아니면 공백             |
| A7  | 홈택스 ID     | X   | 20  | 41  | 홈택스 ID                    |
| A8  | 세무프로그램 코드  | X   | 4   | 45  | `9000`                    |
| A9  | 사업자등록번호    | X   | 10  | 55  | 제출자 사업자번호                 |
| A10 | 법인명(상호)    | X   | 30  | 85  | 상호                        |
| A11 | 담당자 부서     | X   | 30  | 115 | 기본 `경영`                   |
| A12 | 담당자 성명     | X   | 30  | 145 | 고객명                       |
| A13 | 담당자 전화번호   | X   | 15  | 160 | 고객 전화번호                   |
| A14 | 지급자 수      | 9   | 5   | 165 | B 레코드 수                   |
| A15 | 공란         | X   | 5   | 170 | 공백                        |

| 순번 | 필드 | 타입 | 길이 | 누적 | 값/생성 규칙 |

| C1 | 레코드 구분 | X | 1 | 1 | `C` |
| C2 | 자료구분 | 9 | 2 | 3 | `50` |

| C13 | 공란 | X | 4 | 87 | 공백 |
| C14 | 지급액 | 9 | 13 | 100 | 세전금액 |

## 계산/검증 규칙

집계:

```text
A14 = B 레코드 수
B11 = 해당 지급자의 C 레코드 수
B12 = 해당 지급자의 C14 합계
```

소득자 레코드 생성 단위:

```text
지급자별 + 귀속연도 + 귀속월 + 소득자 + 업종코드별로 C 레코드를 생성한다.
같은 소득자에게 두 종류의 업종으로 각각 지급하면 업종코드별로 C 레코드를 2건 생성한다.
이 경우 B11 소득인원도 2명으로 계산한다.
```

세율:

```text
source 기획서 MVP: 일반 사업소득 3%
전산매체 원문: 업종코드별 예외 가능
```

날짜:

```text
C12 귀속연도는 B9 지급연도보다 클 수 없다.
B9 == C12 이고 C18 귀속월 > B13 지급월이면 오류.
```

## 입력 모델 초안

```ts
interface BusinessIncomeSimpleStatementInput {
  submitter: SubmitterInput;
  payer: PayerInput;
  paymentYear: string; // YYYY
  paymentMonth: string; // MM
  paymentHalf: '1' | '2';
  recipients: BusinessIncomeRecipient[];
}

interface BusinessIncomeRecipient {
  industryCode: string;
  name: string;
  identityNo: string;
  foreignerYn: '1' | '9';
  attributionYear: string;
  attributionMonth: string;
  grossAmount: number;
  taxRate: 3 | 5 | 20;
  incomeTax: number;
  localIncomeTax: number;
}
```
