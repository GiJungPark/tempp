# 전자파일 생성 규격 전체

이 문서는 예전 통합 초안의 위치를 유지하기 위한 안내 문서다.

전자파일 생성 규격은 업무별 문서를 단일 기준으로 본다. 통합 문서에 레코드 표를 중복으로 두면 원문 반영 시 서로 다른 내용이 남을 수 있으므로, 구현 시에는 아래 문서를 직접 읽는다.

관련 문서: [[docs/전자자료/index|전자자료 인덱스]], [[docs/원문자료/원문 대조 결과]]

## 구현 기준 문서

| 목표 | 기준 문서 |
| --- | --- |
| 공통 writer, padding, byte length 검증 | [[docs/전자자료/공통 전자파일 생성 규칙]] |
| 원천세 전자파일 생성 | [[docs/전자자료/원천세/원천세 신고 전산매체]] |
| 사업소득 간이지급명세서 생성 | [[docs/전자자료/간이지급명세서/사업소득 간이지급명세서 전산매체]] |
| 기타소득 간이지급명세서 생성 | [[docs/전자자료/간이지급명세서/기타소득 간이지급명세서 전산매체]] |
| 연간 지급명세서 원문 위치와 보류 상태 | [[docs/전자자료/지급명세서/index]] |

## 원문 기준

필드 길이, 레코드 구조, 파일명, 코드표는 `source/전산매체`의 원문 제출요령을 우선한다.

기획서와 사용자가 정리한 표는 서비스 입력값을 전산매체 필드에 매핑하는 보조 자료로 사용한다.

## 주의

- 이 파일은 레거시 링크 호환용 목차다.
- 레코드 표나 계산 규칙을 이 파일에 다시 추가하지 않는다.
- 새 규칙을 발견하면 업무별 문서와 [[docs/원문자료/원문 대조 결과]]에 반영한다.
