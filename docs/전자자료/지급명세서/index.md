# 지급명세서 전자자료 인덱스

사업소득/기타소득 연간 지급명세서 전산매체 원문 위치와 구현 상태를 정리한다.

상위 문서: [[docs/전자자료/index|전자자료 인덱스]], [[docs/원문자료/index|원문자료 인덱스]]

## 현재 상태

연간 지급명세서 원문과 일부 샘플은 `source/`에 보관되어 있다. 아직 `docs/전자자료`에는 월간 원천세/간이지급명세서처럼 구현용 필드 문서를 분리하지 않았다.

월간 MVP 구현 범위:

```text
원천세 신고
간이지급명세서(사업소득)
간이지급명세서(기타소득)
```

연간 지급명세서는 다음 확장 범위:

```text
사업소득 지급명세서
기타소득 지급명세서
```

## 원문에서 확인한 구조

사업소득 지급명세서:

```text
자료구분: 24
레코드: A/B/C
레코드 길이: 190 byte
D 레코드: 없음
C 생성 단위: 소득자별 + 소득귀속연도별 + 지급연도별 + 업종구분별 + 세율별
```

기타소득 지급명세서:

```text
자료구분: 23
레코드: A/B/C/D
레코드 길이: 300 byte
D 레코드: 소득구분 64(서화/골동품 양도소득)인 경우에만 생성
서비스 지원 코드 76/79에서는 D 레코드 생성하지 않음
```

## 구현 시 주의

- 간이지급명세서 제출분은 지급명세서 제출로 간주될 수 있으나, 수정신고 대응을 위해 연간 지급명세서 제출 정책은 별도로 결정해야 한다.
- 지급명세서는 월간 간이지급명세서와 레코드 구조, 자료구분, 집계 기준이 다르다.
- 원문 HWP 기준으로 필드 길이, 레코드 순서, 파일명 규칙을 별도 추출해야 한다.
