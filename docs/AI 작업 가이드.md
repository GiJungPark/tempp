# AI 작업 가이드

AI에게 이 프로젝트 문서를 읽고 작업시키려면 아래 순서로 읽히면 된다.

## Vault 기준

옵시디언 vault root는 프로젝트 루트다.

```text
/Users/nox/Documents/홈택스
```

## 홈택스 스크래핑/API 작업

구현 위치:

- 홈택스 외부 연동, 세션 캐시, RAON 업로드, wqAction 호출은 `src/hometax-scraping`에 둔다.
- 전자파일 생성기와 국세청 전산매체 고정폭 생성 로직은 `src/hometax`에 둔다.
- 다른 모듈이 쿠키, OACX token, `sessionMap`, NTS payload, RAON command를 직접 만지지 않게 한다.
- `src/hometax-scraping/interfaces`의 controller는 개발용 수동 검증 API이며 `/dev/hometax/...` prefix를 사용한다.

먼저 읽을 문서:

1. [[docs/스크래핑/홈택스 스크래핑 구현 스펙|홈택스 스크래핑 구현 스펙]]
2. [[docs/스크래핑/원천세 API 분석|원천세 API 분석]]
3. [[docs/스크래핑/간이지급명세서 API 분석|간이지급명세서 API 분석]]

지시 예:

```text
docs/홈택스 문서 지도.md와 docs/스크래핑/홈택스 스크래핑 구현 스펙.md를 먼저 읽고,
원천세 파일 검증 API를 구현해줘.
```

기획 표는 서비스 입력값 매핑 자료이고, 전산매체 원문 규격을 우선해줘.

## 주의할 점

- `index.md` 파일은 여러 폴더에 있으므로 Obsidian 링크는 `docs/...` 또는 `source/...` 경로 포함 링크를 사용한다.
- 전자자료 생성 규격의 기준은 국세청 전산매체 제출요령 원문이다.
- 사용자가 정리한 표는 서비스 입력값 매핑 자료다.
- 소득자 정보는 신고/전자파일 생성 시점의 일회성 입력값이며, 주소록/마스터로 저장하거나 다음 신고에 재사용하지 않는다.
- `source/`는 원문 보관소이고 `docs/`는 구현 요약 문서다.
- 간이지급명세서 API 일부 코드값은 HAR 추가 확인이 필요하다.
