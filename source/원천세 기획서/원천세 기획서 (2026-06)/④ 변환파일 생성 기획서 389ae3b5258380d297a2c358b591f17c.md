# 공통 규격

- 파일구조: **LINE SEQUENTIAL** (레코드마다 CR/LF), 전환방식 ASCII, 한글 **2byte 완성형 KSC-5601**
- **고정길이 레코드** — 레코드별 정해진 byte 수 정확히 채움(부족분 공란 또는 0)
- 문자 X(n): 좌측정렬, 우측 공란채움 / 숫자 9(n): 우측정렬, 좌측 0채움, 부호없음(원천이행상황 일부 항목은 음수 '-' 허용)
- 주민번호·사업자번호: 하이픈 제거. 외국문자·영문명은 소리나는 대로 한글 표기
- 소득자 식별: 내국인=주민등록번호, 외국인 거주자=외국인등록번호(없으면 여권번호+국적코드). 전산매체 C레코드의 내외국인 구분 필드에 반영
- 날짜: YYYYMMDD / 연월 YYYYMM

# 신고구분상세코드 (파일명 확장자)

01 정기 / 02 수정 / 03 기한후 (★ 02 수정은 현재 미지원 — ① 공통 기획서 참조)

- **자동 분기 (마감일 D = 지급월 다음달 10일, 휴일이면 다음 영업일로 순연)**: 직전 달 지급분을 **1일부터 D까지** 신고 → 정기(01). **D 다음날부터 말일**의 직전 달 기한후, 또는 과거 달 지급분 → 기한후(03). 1일부터 D까지(정기 기간)에는 기한후를 받지 않음. 10일/11일로 못박지 말 것. (운영 정책은 ① 공통 기획서)

# ★ 귀속연월·지급연월·제출연월 규칙 (귀속≠지급 지원의 근간)

![image.png](%E2%91%A3%20%EB%B3%80%ED%99%98%ED%8C%8C%EC%9D%BC%20%EC%83%9D%EC%84%B1%20%EA%B8%B0%ED%9A%8D%EC%84%9C/image.png)

그림의 규칙·신고월·가드레일을 홈택스 형식검증 관점에서 정확히 정의. 입력 밸리데이션 기준. 위반 시 반려되므로 입력 단계에서 선제차단.

- **① 지급연월 = 제출연월 − 1 (강제)** — 전산매체 매뉴얼 작성법 확인 완료. 여기서 **'제출연월'은 그 신고서의 귀속 제출월(= 지급연월+1)**이지 실제 업로드한 달력 월이 아니다. → **과거 놓친 달도 신고 가능**: 그 달 신고서를 제출연월=지급연월+1로 만들어 기한후(03)로 올린다(업로드 시점은 나중이어도 됨). 2015년 3월분부터 지급연월=제출연월인 자료는 불가.
- **② 지급연월 ≥ 귀속연월** (귀속보다 먼저 지급 불가)
- **③ 같은 해 안에서는** 귀속월 이후 어느 달이든 지급월로 허용 (예: 3월 귀속 → 4월에서 12월 지급 정상)
- **④ 지급월 상한 — 기획 가드레일 (세법상 지급간주 아님)**: 일반 사업·기타소득은 세법상 지급시기 의제가 없어(§144① · §145① 모두 "지급할 때" 원천징수) 법적 지급 상한이 없다. 다만 무한정 늦은 지급(예: 12월 귀속·이듬해 7월 지급)을 막기 위해 **서비스 정책상 입력 한도**를 둔다 (근로소득 §135 패턴 준용):
    - 1월에서 11월 귀속 → 같은 해 **12월 지급**까지
    - 12월 귀속 → **다음해 2월 말일** 지급까지
    - 이 범위를 벗어나는 지급월은 입력 단계에서 차단. (세법 의제가 아니라 서비스 가드레일임을 UI 안내에 명시)
- 참고: §144의5(연말정산 사업소득)·§145의2(법인세법 §67 처분 기타소득)에는 다음해 2월말 지급간주가 있으나, 둘 다 본 서비스 범위 밖.

# 집계 무결성 검증

![image.png](%E2%91%A3%20%EB%B3%80%ED%99%98%ED%8C%8C%EC%9D%BC%20%EC%83%9D%EC%84%B1%20%EA%B8%B0%ED%9A%8D%EC%84%9C/image%201.png)

- 수식(그림 아래 상세): 일치① `A.지급자수 = B 레코드 수` / 일치② `B.소득인원 = C 레코드 수` / 일치③ `B.총지급액 = Σ C.지급액` (셋 다 맞아야 형식검증 통과, 하나라도 어긋나면 반려)
- 원천징수이행상황신고서: 본표 인원·총지급액·세액이 간이지급명세서·지급명세서 합계와 정합

# 코드 유효성 검증

- 코드는 적용시작·종료일자가 있음 → 지급·귀속연월 기준 유효성 체크
- 예: 부표 소득코드 CA6·CA7·CA8은 2026.5.16부터 적용(우리 서비스 범위 밖). 사업 업종코드·기타 소득구분코드도 동일

# 0원 처리 반영

- 소액부징수(원천세 1,000원 미만): 원천세 0 기록, 지급명세서 제출 시 소액부징수 건수 집계
- 과세최저한(기타 5만원 이하): 원천세·지방세 0 기록

# 형식검증 대응·자체검증

- 제출 직전 홈택스 **형식검증**이 관문 — byte 레이아웃 오류·집계 무결성 위반 시 반려
- 변환파일 생성 단계에서 자체 검증 선행(길이·CRLF·인코딩·집계) — validator 도구 활용

# 버전 관리

- 서식·전산매체는 매년 개정 → 귀속연도별 레이아웃 버전 관리. 개발 착수 시 홈택스 자료실 최신본 재확인
- 보유본 현황: 원천징수이행상황신고서 2026.5.16 / 간이지급명세서 2024.01.01 / 지급명세서 2025.12.18 (지급명세서 하반기 신본 대기)

# 전자신고 제출 절차

1. 변환파일 생성 → 2. 홈택스 로그인(인증서/간편인증) → 3. 파일 변환신고(회계프로그램) 메뉴 → 4. 업로드+암호 → 형식검증 → 5. 제출 → 6. 접수증 조회
