# ① 원천세 공통 기획서

> **편집 원칙: 이 문서에는 모든 소득유형(사업·기타, 추후 근로·퇴직 등)에 공통으로 적용되는 것만 둔다. 한 소득유형에만 해당하는 내용은 해당 소득 기획서(②③)에 둔다.** 소득유형별 상세는 [② 사업소득], [③ 기타소득] 기획서, 변환 규칙은 [④ 변환파일 생성] 기획서 참조.
> 

# 목적

사업·기타소득을 지급하는 개인사업자 고객이 **원천세 신고·간이지급명세서·지급명세서 제출**을 편하고 정확하게 하도록 돕는다.

# 용어

- **세전금액**: 원천세를 공제하기 전 지급해야 하는 금액
- **세후금액**: 원천세를 공제한 뒤 실제 지급하는 금액
- **소득자**: 고객이 용역의 대가를 지급하는 상대방
- **귀속월**: 소득자가 용역을 제공한(일한) 달 — "언제 일했나"
- **지급월**: 지급자가 실제로 대가를 준 달 — "언제 돈 줬나". 원천징수·신고기한의 기준.
- **제출월 (제출연월)**: 원천징수이행상황신고서에 기재하는 값으로 **지급월 + 1** (전산매체 강제규칙: 지급연월 = 제출연월 − 1). 신고서가 귀속되는 제출 월이며, **실제 홈택스에 올린 날짜가 아니다.**
- **신고월 (= 앱 진입월)**: 고객이 앱에 들어와 신고서를 올린 달력상의 달. 이 달의 며칠인지가 마감일(D) 이내면 정기신고, 지나면 기한후신고. 입력 모델의 "진입월"이 곧 신고월이다. (과거 놓친 분은 신고월이 제출월보다 늦다)
- **원천세**: 사업소득은 지급금액×3%, 기타소득은 (소득금액)×20%
- **지방세**: 국세(원천세)의 10%
- **정기신고 마감일(D)**: 지급월의 다음달 **10일**. 단 D가 토요일·일요일·공휴일(근로자의 날 포함)이면 **그 다음 영업일로 순연**(국세기본법 §5). 공휴일이 이어지면 그 이후 첫 영업일. → D는 보통 10일이지만 휴일이면 11일·12일 등으로 밀릴 수 있다.

**네 가지 '월' 한눈에 (예시)**

| 상황 | 귀속월(일한 달) | 지급월(돈 준 달) | 제출월(신고서 기재=지급+1) | 신고월(실제 올린 달) | 모드 |
| --- | --- | --- | --- | --- | --- |
| 귀속=지급 | 4월 | 4월 | 5월 | 5월 5일 | 정기(01) |
| 귀속≠지급 | 3월 | 4월 | 5월 | 5월 5일 | 정기(01) |
| 직전 달 기한후 | 4월 | 4월 | 5월 | 5월 20일 | 기한후(03) |
| 과거 누락분 | 2월 | 2월 | 3월 | 5월 20일 | 기한후(03) |

# 정책

## 대상

- 사업소득 또는 기타소득을 지급하는 개인사업자

## ★ 귀속월·지급월 (핵심)

- **귀속월과 지급월이 달라도 신고한다.** (기존 기획에서는 제외 대상이었으나 이제 지원)
- **귀속월·지급월 모두 고객이 직접 입력·수정한다(항상 열림, 특정 기간/화면에 한쪽을 고정하지 않음).** 고객이 둘을 바꾸면 **앱이 (지급월·오늘 날짜)로 정기신고/기한후신고를 자동 판정**하고, 기한후면 가산세를 계산한다. 날짜 구간으로 입력을 막는 분기는 두지 않는다. (2026-06-24 이안·에버 결정)
- 입력 시 귀속연월과 지급연월을 **각각 분리 입력**받는다. 둘이 같으면 동일값을 그대로 쓴다.
- **신고·제출 시점은 지급월이 정한다.** 귀속연월은 명세서에 소득 귀속연도로 기재되어 소득자의 종합소득세·연말정산 기준연도를 정한다.
- 연도 걸침(예: 12월 귀속·익년 1월 지급)은 귀속연월을 실제 일한 시점으로 정확히 기재. (연월 검증 규칙은 ④ 변환파일 기획서)
- **지급월 상한(기획 가드레일)**: 일반 사업·기타소득은 세법상 지급시기 의제가 없으나(실제 지급 시 원천징수), 너무 늦은 지급을 막기 위해 **1월에서 11월 귀속은 같은 해 12월 지급까지, 12월 귀속은 다음해 2월 말일 지급까지**만 입력 허용(세법 의제 아니라 서비스 정책). 상세는 ④ 변환파일 기획서.

## 입력 화면 동작 (지급월·귀속월 선택)

모드는 **선택한 지급월의 마감일 D를 오늘과 비교**해 자동 판정 (D = 지급월 다음달 10일, 휴일이면 순연):

| 구분 | 정기신고 (01) | 기한후신고 (03) |
| --- | --- | --- |
| 판정 | 오늘 ≤ 해당 지급월 D | 오늘 > 해당 지급월 D |
| 전형적 대상 | 직전 달 지급분 | 과거 누락분(직전 달도 D 경과 후 포함) |
| 가산세 | 없음 | 원천세 납부지연분 |
| 비고 | 마감 전엔 덮어쓰기 재신고 | 놓친 달 늦은 신고 지원 |

예) 진입 2027년 5월 → 지급 4월은 5월 10일까지 정기(01)·이후 기한후(03); 지급 2월(과거 누락분)은 D(3월 10일) 경과로 5월 내내 기한후(03)이며 1일부터 10일 사이에도 신고 가능

- 진입 시 **지급월 기본값 = 진입한 달(= 신고월)의 전월** (가장 흔한 직전 달 신고). 고객이 바꾸지 않으면 이 값으로 신고.
- **지급월·귀속월 모두 고객이 자유롭게 수정**(특정 기간에 고정하지 않음). 지급월은 과거 달로 바꿔 놓친 달의 늦은 신고(기한후)를 지원한다. 단 **미래(진입한 달 또는 그 이후)는 선택 불가** → 지급월 ≤ 진입한 달의 전월.
- **신고 모드는 선택한 지급월의 마감일 D를 오늘과 비교해 자동 판정**(D = 그 지급월 다음달 10일, 휴일이면 순연):
    - **오늘 ≤ 해당 지급월의 D**: **정기신고(신고구분 01)**, 가산세 없음.
    - **오늘 > 해당 지급월의 D**: **기한후신고(신고구분 03)**, 원천세 납부지연 가산세.
- 날짜 구간(1일부터 D까지 등)으로 과거분 기한후 입력을 **막지 않는다** — 각 지급월의 D로 건별 판정하므로, 예) 3월 1일에 1월 지급분(D=2월 10일 경과)을 기한후로 신고할 수 있다.
- **귀속월은 선택한 지급월 기준으로 정해진다** (지급연도 = Yp년, 전년 = Yp−1년):

| 선택한 지급월 | 귀속월 선택지 |
| --- | --- |
| Yp년 3월에서 12월 | Yp년 1월부터 지급월까지 |
| Yp년 2월 | Yp년 1월·2월, 또는 **Yp−1년 12월** |
| Yp년 1월 | Yp년 1월, 또는 **Yp−1년 12월** |
- (귀속월은 항상 지급월 이하 + "12월 귀속은 다음해 2월 말일 지급까지" 가드레일이 반영된 것)

**예시 (진입 2027년 5월)**

- 지급 **2027년 4월** 선택: D = 5월 10일 → 5월 10일까지 정기(01), 5월 11일부터 기한후(03). 귀속은 **2027년 1월에서 4월** 중 선택.
- 지급 **2027년 2월**(과거 누락분) 선택: D(3월 10일) 이미 경과 → 5월 어느 날이든 기한후(03)·가산세. **1일부터 10일 사이에도 입력·신고 가능.**
- 미래 불가: 5월에 지급월을 **5월 이후로는 못 고름**(아직 신고기한 전) → 지급월 ≤ 4월.

## 신고·제출 일정 (모두 지급월 기준)

| 산출물 | 마감 | 비고 |
| --- | --- | --- |
| 원천세 신고·납부 | 지급월의 다음달 10일 (= 마감일 D) | 휴일이면 다음 영업일로 순연 |
| 간이지급명세서 | 지급월의 다음달 말일 | 휴일이면 다음날. 동의 시 원천세 신고후 즉시 제출, 미제출자는 지급월 다음달 마지막주 첫 평일 알림 |
| 지급명세서 | 사업소득: 다음 연도 3월 10일 / 기타소득: 다음 연도 2월 말일 | 휴일이면 다음날. 간이지급명세서 제출시 생략 가능하나 수정신고 대비 제출이 안전. 다음 연도 2/1 알림 |

## 서비스 운영 시점 / 신고 모드

서비스는 매월 1일부터 말일까지 항상 열린다. 신고 모드는 기간 게이트가 아니라 **선택한 지급월의 마감일 D를 오늘과 비교**해 건별로 갈린다(D = 지급월 다음달 10일, 휴일이면 순연).

| 판정 | 처리 대상 | 신고구분 | 가산세 |
| --- | --- | --- | --- |
| 오늘 ≤ 해당 지급월 D | 그 지급월 지급분 정기신고 | 01 | 없음 |
| 오늘 > 해당 지급월 D | 그 지급월 지급분 기한후(과거 누락분 포함) | 03 | 원천세 납부지연분 |
- 과거 누락분 기한후는 **기간 제한 없이 아무 때나** 입력·신고 가능(1일부터 D 사이에도). 날짜 구간으로 기한후를 막지 않는다. (2026-06-24 결정)
- 간이지급명세서는 해당 지급월 다음달 말일까지가 기한.
- 연간 지급명세서(다음해 2월 말일)는 이 월간 사이클과 별개로 처리.
- 수정신고(신고구분 02)는 미지원(확정). 정기·기한후만 처리.

## 지방소득세 (위택스) 연계

- 원천세는 **국세(홈택스)**, 지방소득세 특별징수분은 **지방세(위택스)** — 기관이 달라 둘 다 신고해야 신고가 끝난다.
- 흐름: 홈택스에서 원천세 신고 완료 → 홈택스가 띄우는 **“지방소득세 신고(위택스)” 바로가기** 버튼으로 연계 → 위택스에서 특별징수 지방소득세 신고·납부.
- 지방소득세액 = 원천세(소득세)의 10%로 이미 산출됨 → 위택스 신고에 필요한 **소득자별·합계 금액을 그대로 안내**한다.
- 위택스 접수증·납부서도 앱에서 불러와 다운로드(공통 기능).
- **주의 안내**: 원천세(홈택스)만 하고 지방소득세(위택스)를 빠뜨리면 지방세도 가산세 → 앱이 “지방소득세 신고까지 완료” 여부를 체크·안내한다.

## 재신고·기한후 신고

- 마감일 D 전이면 다시 신고 시 최신 신고서로 덮어쓴다.
- D 다음날부터는 기한후 신고 — 가산세 포함(납부세액 0이면 가산세 없음).
- **가산세 (사업·기타 공통 — 원천세 납부지연)**: 미납세액의 3% + 미납세액 × 미납일수 × **22/100,000**(1일 10만분의 22 — 국세기본법 시행령 §27조의4), 한도는 미납세액의 50%(국세기본법 §47조의5). **원천세는 무신고 가산세가 없다**(납부지연 가산세만). 사업소득·기타소득 동일 적용.

## 원천세가 0원이 되는 경우 (소액부징수·과세최저한)

- **소액부징수 (모든 소득 공통)**: 소득자별 원천세가 1,000원 미만이면 징수하지 않는다(0원) — 소득세법 §86. 사업소득은 과세최저한이 없어 소액 지급(예: 3만원 × 3% = 900원)에서 자주 발생. 기타소득은 과세최저한이 먼저 걸려 원천세가 0원이거나 최소 1만원 이상이라 사실상 미발생. (지급명세서 제출 시 소액부징수 건수 집계)
- **과세최저한**: 기타소득금액이 건별 5만원 이하면 0원(§84③). 기타소득 전용 — 상세는 ③ 기타소득 기획서.
- **원천세 0원인 소득자도 신고서에 포함**: 소액부징수·과세최저한으로 원천세가 0원이 된 소득자도 원천징수이행상황신고서·지급명세서에 인원·총지급액으로 포함한다(누락 금지). 사업·기타 공통.

## 고객에게 제공하는 문서

월 지급대상 / 소득자별 지급명세서 / 국세 접수증 / 원천징수이행상황신고서 / 국세 납부서 / 지방세 접수증 / 지방세 납부서

## 공통 데이터모델 (DB 저장)

소득구분(업종코드 또는 소득구분코드), 소득자 이름, 외국인 여부(내국인 1 / 외국인 9), 식별번호(내국인=주민등록번호 / 외국인=외국인등록번호 또는 여권번호), **귀속월, 지급월**, 세전금액, (필요경비·소득금액), 원천세, 지방세, 세후금액

- 소득자: 가장 최근 입력 데이터 / 세전금액·세후금액: 가장 최근 선택 유지
- 계좌·핸드폰번호는 원천세 신고·명세서 제출에 불필요 → 수집하지 않는다
- **외국인 거주자 소득자 처리** (외국인이 사업·기타속득을 받는 경우 — 사업·기타 공통):
    - **대상은 거주자만.** 전산매체 거주자구분 = `1`(거주자) 고정. 비거주자(거주자구분 2)는 사업·기타 서식에서 오류이며 본 서비스 범위 밖 → 입력 차단. (근로·일용·퇴직소득도 범위 밖)
    - **외국인 여부**를 입력받아 전산매체 **내외국인구분**에 반영: 내국인 = `1`, 외국인 = `9`. (미기재·1/9 외 값은 전산매체 오류)
    - **식별번호(13자리 주민등록번호 필드)**: 내국인 = 주민등록번호 / 외국인 = **외국인등록번호**, 없으면 **여권번호**. (전산매체는 이 필드에 주민등록번호·외국인등록번호·여권번호를 모두 허용 — 관련서류와 번호가 일치하면 정상 처리)
    - **국적코드·거주지국코드·단일세율은 입력·저장하지 않는다** — 사업·기타속득 서식에 해당 필드가 없다(근로소득 지급명세서 전용 = 범위 밖). 외국인 거주자라도 사업소득 3%·기타속득 20% 동일 적용.
    - 외국인등록번호·여권번호도 주민등록번호와 동일하게 **암호화 저장**. 전산매체 byte 배치·검증은 ④ 변환파일 기획서.
    - **국적코드 없음은 간이지급명세서 과 연간 지급명세서 모두 동일** — 사업·기타 4개 서식 전부 국적코드 필드 자체가 없다(국적코드는 근로소득 서식 전용).
    - 근거: ① 사업소득 지급명세서(거주사업) 거주자구분 C10(=1)·내외국인구분 C11(1/9)·식별번호 C6(주민·외국인등록·여권 허용) ② 기타속득 지급명세서(거주기타) 거주구분 C8(=1)·내·외국인구분 C9(1/9)·식별번호 C6 ③ 간이지급명세서(사업·기타)도 동일 구조. 4개 서식 모두 국적코드 없음.

## 공통 기능

- 세전금액/세후금액 입력을 고객이 선택. 세전금액 입력 → 원천징수세액(지방세 포함)·세후금액 계산 / 세후금액 입력 → 세액·세전금액 역산.
- 지급내역서는 공유 링크로 전달(사장님이 카톡 등으로 소득자에게 전달; 전화번호 미수집).
- 신고후 홈택스·위택스에서 접수증·이행상황신고서·납부서를 불러와 앱에서 다운로드.
- 납부확인 버튼 → 인증 후 납부내역 표시. 1년치 세전금액·기납부세액 문서 제공.

## 편의 기능 (반복·일괄 신고)

### 소득자 주소록 + 지난달 불러오기

- 소득자를 한 번 등록하면 **주소록(소득자 마스터)**에 저장: 이름·외국인 여부·주민등록번호(외국인은 외국인등록번호)·소득유형(업종코드 또는 소득구분코드)·세전/세후 기본 선택. (계좌·핸드폰은 신고에 불필요하여 수집하지 않음)
- 매달 신고 시작 시 **"지난달 신고분 그대로 불러오기"** → 소득자 명단·소득유형이 자동으로 채워지고, 금액만 수정하면 된다.
- 진입(1일부터 D까지) 첫 화면 = "이번 달 신고 후보"(지난달 명단 기반). 빠진 사람 추가·제외 토글.
- 소득자 검색·즐겨찾기. 주민등록번호 등 민감정보는 암호화 저장(보안 정책 준수).

### 여러 명 한 번에 입력 (일괄)

- **표(grid) 입력**: 한 화면에서 여러 소득자 행을 동시에 입력·수정. 각 행 금액 입력 시 세액·반대금액 자동 계산.
- **엑셀/CSV 업로드**: 이름·주민등록번호·금액·소득유형 열을 자동 매핑해 일괄 등록.
- 업로드·입력 시 **검증**: 주민등록번호 형식, 코드 유효성(지급·귀속연월 기준), 소득자 중복.
- 상단에 **합계 요약**(인원·총지급액·총원천세·총세후) 실시간 표시.
- (2차 확장) 카톡·메모에서 복사한 명단 **붙여넣기 파싱**.

### 입력 화면 연결 (소득유형별 그리드)

주소록·일괄 입력은 ②③의 단건 입력·계산 규칙을 다건으로 실행하는 것. 화면은 소득유형으로 나뉴다.

- **지급월은 화면 상단에서 한 번 선택**(기본=직전 달, 과거 변경 가능) → 그 신고 세션의 모든 행에 공통 적용. **귀속연월은 행별로 선택**(지급월 기준 선택지 규칙; 기본=지급월과 동일).
- 화면을 **사업소득 / 기타소득 두 구획(탭)**으로 분리 — 유형마다 컬럼·계산이 다르므로.

**사업소득 그리드 컬럼**: 소득자명 · 외국인여부 · 주민등록번호 · 업종코드 · 귀속연월 · [세전/세후 택1] 입력금액 → (자동) 원천세 3% · 지방세 · 반대금액

**기타소득 그리드 컬럼**: 소득자명 · 외국인여부 · 주민등록번호 · 소득구분코드(76·79) · 귀속연월 · [세전/세후 택1] 입력금액 → (자동) 필요경비 60% · 소득금액 · 원천세 20% · 지방세 · 반대금액

- 세전/세후 기준은 **그리드(유형) 단위 토글**이 기본, 행별 예외 허용.
- 0원 자동 표시: 소액부징수(원천세 1,000원 미만)·과세최저한(기타 5만원 이하) 해당 행은 원천세 0 + 배지.
- 하단 **합계**: 유형별 + 전체(인원·총지급액·총원천세·총세후).

**입력 경로 3가지가 같은 그리드로 수렴**

- (a) 주소록 "지난달 불러오기" → 저장된 소득유형대로 각 그리드에 자동 배치
- (b) 그리드에서 행 직접 추가
- (c) 엑셀/CSV 업로드 → 소득유형 열로 자동 분류 후 채움

**검증(행/교차)**: 주민번호 형식 · 코드 유효성(지급·귀속연월 기준) · 금액 양수 · 같은 소득자+같은 귀속/지급월 중복 경고.

- (설계 선택) 사업·기타를 한 그리드에 섞고 행별 소득유형 드롭다운으로 분기하는 안도 가능하나, 컬럼이 달라 **유형별 구획을 기본**으로 한다.

## 제공하지 않는 서비스 (공통)

- 원천세 반기 신고
- 소득유형별 제외는 각 소득 기획서 참조
